##  UX Project Scenario 2 – Healthcare Website
A mobile health app helps users track daily exercise, water intake, sleep, and medication reminders.

1. Identifying and Writing Problem Statements
User Pain Points
Users forget to take medicines on time.
Health data is scattered across different apps.
Elderly users find navigation confusing.
Users lose motivation because progress tracking is unclear.
UX Problem Statements

“Busy working adults need a way to receive simple and timely medication reminders because they often forget schedules during work hours, which results in missed doses and health risks.”

“Elderly users need a way to navigate health tracking features easily because the app interface is complex, which results in frustration and reduced app usage.”

2. Identifying Appropriate Research Methods
Research Methods Chosen
a) User Interviews

Why?
Interviews help understand users’ daily health routines, challenges, and emotional experiences with health apps.

b) Usability Testing

Why?
Testing allows designers to observe how users interact with the app and identify navigation difficulties.

c) Surveys

Why?
Surveys collect feedback from many users quickly regarding features, satisfaction, and expectations.

## UX Project Scenario 2 – E-Commerce Website
Project Overview

An online shopping website for electronics, clothing, and household products.

1. Identifying and Writing Problem Statements
User Pain Points
Difficult product search and filtering
Complicated checkout process
Lack of trust in product reviews
Slow mobile website experience
UX Problem Statements

“Online shoppers need a way to filter products quickly because too many irrelevant search results appear, which results in wasted time and abandoned purchases.”

“Mobile users need a faster checkout process because lengthy forms and multiple steps create frustration, which results in cart abandonment.”

2. Identifying Appropriate Research Methods
Research Methods Chosen
a) Usability Testing

Why?
Helps identify where users struggle during product search and checkout.

b) Analytics Review

Why?
Website analytics reveal drop-off points, cart abandonment rates, and user behavior patterns.

c) Surveys

Why?
Collects customer opinions about product search, trust, and checkout experience.
